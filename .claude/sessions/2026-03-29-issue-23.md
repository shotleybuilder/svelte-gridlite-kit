## Decisions

- **Monorepo**: pnpm workspaces, single repo, shared test infra, atomic commits
- **Breaking changes OK**: single consumer (us), 0.x beta вҖ” no deprecation shims or backwards-compat props
- **Both adapters are separate packages**: PGLite (~3MB WASM) and TanStack DB are both heavy deps; core stays dependency-free
- **`db`/`table`/`source` props removed**: replaced by `adapter` prop, clean cut

## Phases

### Phase 1: Monorepo setup + adapter interfaces + PGLite adapter extraction (DONE)
- Convert to pnpm workspaces monorepo (`packages/core`, `packages/pglite`)
- Define `QueryAdapter`, `LiveQueryHandle`, `SuggestionProvider`, `SchemaProvider` in core types
- Extract all PGLite-specific code into `packages/pglite`
- Remove `db`/`table`/`source` props from GridLite, replace with `adapter` prop
- Move `@electric-sql/pglite` from core dep to pglite adapter peer dep
- Tests: all existing tests pass under new structure, add adapter-level tests

### Phase 2: Decouple FilterBar/FilterGroup from PGLite (DONE)
- Remove `db` prop from FilterBar and FilterGroup
- Replace direct `db.query()` calls with `adapter.getSuggestions()` / `adapter.getNumericRange()`
- Thread adapter through GridLite вҶ’ FilterBar вҶ’ FilterGroup
- Consolidate duplicate suggestion code between FilterBar and FilterGroup

### Phase 3: Decouple state persistence (migrations + views) (DONE)
- Added 6 view CRUD methods to QueryAdapter interface (loadViews, saveView, deleteView, loadColumnState, saveColumnState, loadActiveView)
- PGLite adapter implements all 6 using existing views.ts
- Tests: 5 new adapter-level persistence tests

### Phase 1AвҖ“1E: Refactor QueryAdapter from SQL to Structured IR (DONE)

Discovered during Phase 4 planning: the QueryAdapter interface was SQL-native (`createLiveQuery(sql, params)`, `execute(sql, params)`), making it impossible for non-SQL backends like TanStack DB. Refactored to pass structured descriptors instead of SQL strings.

#### Phase 1A: New descriptor types + updated interface
- Added 5 descriptor types: `QueryDescriptor`, `CountDescriptor`, `GroupSummaryDescriptor`, `GroupCountDescriptor`, `GroupDetailDescriptor`
- Replaced `createLiveQuery(sql, params)` with `createLiveQuery(query: QueryDescriptor)`
- Replaced `execute(sql, params)` with 4 typed methods: `executeCount()`, `executeGroupSummary()`, `executeGroupCount()`, `executeGroupDetail()`
- Removed `getTable()` / `getSource()` вҖ” adapter knows its own source
- Simplified `getDistinctValues()` / `getNumericRange()` вҖ” no options needed, adapter uses its own internals

#### Phase 1B: Refactor GridLite.svelte
- Removed all SQL builder imports and `querySource` assembly
- 5 call sites changed: `rebuildQuery()`, `updateTotalCount()`, `rebuildGroupedQuery()` (summaries + count), `fetchGroupChildren()` (sub-groups + detail)
- GridLite now passes structured descriptors to the adapter

#### Phase 1C: Refactor PGLiteAdapter
- Adapter now imports builder functions and builds SQL internally
- Added `resolveSource()` private helper
- Implemented all 5 new methods, wrapping builder output вҶ’ SQL вҶ’ PGLite execution
- Removed `execute()`, `getTable()`, `getSource()`

#### Phase 1D: Update tests
- Replaced `execute()` tests with `executeCount`, `executeGroupSummary`, `executeGroupCount`, `executeGroupDetail`
- Updated `createLiveQuery` tests to pass descriptors
- Removed `getTable / getSource` tests

#### Phase 1E: Cleanup
- Removed `allowedColumns` prop from FilterBar and FilterGroup (adapter manages this internally)
- Updated session doc

### Phase 4: TanStack DB adapter (DONE)

Created `packages/tanstack-db` вҶ’ `@shotleybuilder/gridlite-adapter-tanstack-db`

#### Files created
- `adapter.ts` вҖ” TanStackDBAdapter class implementing full QueryAdapter
- `index.ts` вҖ” Public exports + createTanStackDBAdapter factory
- `live.ts` вҖ” LiveQueryHandle wrapper around createLiveQueryCollection
- `schema.ts` вҖ” Derive ColumnMetadata from Zod schema
- `storage.ts` вҖ” StorageProvider interface + InMemoryStorage + LocalStorageProvider
- `query-translator.ts` вҖ” Descriptor вҶ’ TanStack DB query builder chain
- `operator-map.ts` вҖ” FilterOperator вҶ’ TanStack DB operator functions

#### Key design decisions
- Uses `createLiveQueryCollection` (framework-agnostic, no Svelte context needed) вҖ” NOT `useLiveQuery`
- One-shot queries use `toArrayWhenReady()` to drive the D2 pipeline (not `onFirstReady` which hangs without subscribers)
- `orderBy()` takes `'asc'`/`'desc'` string as second arg, NOT `{ desc: boolean }`
- `fn.where()` callback receives `{ source: { ...row } }` (matching `.from({ source: coll })` alias)
- JSONB operators (`jsonb_has_key`/`jsonb_not_has_key`) use `fn.where()` fallback since they can't be expressed in D2 expression tree
- `intervalOffset` not supported вҖ” throws descriptive error
- StorageProvider is pluggable (InMemoryStorage default, LocalStorageProvider for browser persistence)
- Peer dep on `@tanstack/db >=0.5.0` only (v0.6.1 installed)

#### Tests: 78 passing

- query-translator.test.ts (23) вҖ” applyFilters, applyGlobalSearch, applySorting, applyPagination, applyJsonbFilters, combined operations
- storage.test.ts (9) вҖ” InMemoryStorage CRUD for column state, views, default view

## Notes
- `db.live.query()` is the hardest coupling вҖ” it's the reactive subscription backbone
- `query/builder.ts` is already pure (no db dependency) вҖ” stays in core, used by PGLite adapter
- FilterBar/FilterGroup have duplicate `db.query()` for suggestions вҖ” consolidate in Phase 2
- View persistence (Phase 3) flagged as optional/later in the issue
- Column-to-column comparison with INTERVAL offset (`valueColumn` + `intervalOffset`) has no TanStack DB equivalent вҖ” will need a custom comparator function in the TanStack DB adapter
- TanStack DB operators available: `eq`, `gt`, `gte`, `lt`, `lte`, `like`, `ilike`, `inArray`, `isNull`, `and`, `or`, `not`, `count`, `avg`, `sum`, `min`, `max`, `upper`, `lower`, `length`, `concat`, `coalesce`
- sertantai-legal is the only consumer вҖ” 3 pages use GridLite: browse, admin/lrt, admin/lat/queue (all on PGLite today)
